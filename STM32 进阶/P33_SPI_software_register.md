# 电路原理

SPI是高速 全双工  串口 通信

需要4根线  : 

 	1. **sclk 时钟线 => 控制时序** 
 	2. **MOSI:数据线  => 主设置发送给从设备**
 	3. **MISO:数据线 => 从设备发送给主设备**
 	4. **CS:片选线 => 选择使用对应的从设备**

SPI统一使用0模式  =>  空间SCK为低电平  第一个跳变沿 发送数据

标准时序图

**(1) 进行片选  CS 拉低**

**(2) 准备MOSI需要发送的数据  =>  高电平优先   => 写数据的时候是先写数据  再移位**

**(3) 拉高时钟线 => 上升沿  => 此时MOSI的数据已经发送出去**

**(4) sck处于高电平的时候  读取MISO的值   =>  读数据的时候是先移位  再读数据**

**(5) 拉低时钟线 => 下降沿**

**(6) 循环处理数据8次**

**(7) 停止片选**

# SPI驱动

(1) 软件模拟  

 	1. CS => PC13  推挽输出
 	2. SCK => PA5 推挽输出
 	3. MISO => PA6 浮空输入
 	4. MOSI => PA7 推挽输出

## W25Q32接口	

(1) 读取ID的标准流程 =>  所有的W25Q32指令的流程

 	1. 开始片选
 	2. 发送执行的指令
 	3. 按照要求发送接收字节
 	4. **关闭片选**

(2) 擦除扇区

 	1. 最小单位 4kb 一个扇区
 	2. 等待忙状态
 	3. 开启写使能
 	4. 开始片选
 	5. 写入指令  0x20
 	6. 传入地址
 	7. 关闭片选
 	8. 关闭写使能

(3) 写入数据

一次只能在一个页中写数据  超出范围会从头覆盖

 	1. 等待忙状态
 	2. 开启写使能
 	3. 开始片选
 	4. 写入指令 0x02
 	5. 传入地址
 	6. 写入数据
 	7. 关闭片选
 	8. 关闭使能

(4) 读取数据

读数据是不受到页的限制  可以一次性读多个页  但是不推荐

 	1. 等待忙状态
 	2. 开始片选
 	3. 写入指令 0x03
 	4. 传入起始地址
 	5. 读取数据
 	6. 关闭片选
