# 电路原理

SPI是高速 全双工  串口 通信

需要4根线  : 

 	1. **sclk 时钟线 => 控制时序** 
 	2. **MOSI:数据线  => 主设置发送给从设备**
 	3. **MISO:数据线 => 从设备发送给主设备**
 	4. **CS:片选线 => 选择使用对应的从设备**

SPI统一使用0模式  =>  空间SCK为低电平  第一个跳变沿 发送数据



# 

(1) CR1: 选择SPI基本配置  数据帧大小  数据帧格式  SPI使能  波特率设置  主设备选择 SPI模式选择   特别: 代码手动拉高拉低片选线  SSM

(2) CR2: 开启中断  开启DMA功能  额外功能 SS输出功能(不使用 )

(3) SR: 当前SPI运行的状态   只需要用到  TXE 和 RXNE

(4) DR: 对应两个数据缓存  对DR写 => 写入到发送缓存  对DR读 => 读取接收缓存

## W25Q32接口	

(1) 读取ID的标准流程 =>  所有的W25Q32指令的流程

 	1. 开始片选
 	2. 发送执行的指令
 	3. 按照要求发送接收字节
 	4. **关闭片选**

(2) 擦除扇区

 	1. 最小单位 4kb 一个扇区
 	2. 等待忙状态
 	3. 开启写使能
 	4. 开始片选
 	5. 写入指令  0x20
 	6. 传入地址
 	7. 关闭片选
 	8. 关闭写使能

(3) 写入数据

一次只能在一个页中写数据  超出范围会从头覆盖

 	1. 等待忙状态
 	2. 开启写使能
 	3. 开始片选
 	4. 写入指令 0x02
 	5. 传入地址
 	6. 写入数据
 	7. 关闭片选
 	8. 关闭使能

(4) 读取数据

读数据是不受到页的限制  可以一次性读多个页  但是不推荐

 	1. 等待忙状态
 	2. 开始片选
 	3. 写入指令 0x03
 	4. 传入起始地址
 	5. 读取数据
 	6. 关闭片选
